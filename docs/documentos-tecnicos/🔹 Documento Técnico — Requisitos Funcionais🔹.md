# 🔹 Documento Técnico — Requisitos Funcionais🔹

## RF-01 — Autenticação
O sistema deve autenticar usuários por e-mail e senha e abrir sessão autenticada.

## RF-02 — Controle por perfil
O sistema deve restringir funcionalidades por perfil (`ADMIN`, `GESTOR`, `ANALISTA`, `OPERACOES`).

## RF-03 — Cadastro de alerta manual
O sistema deve permitir cadastro de alerta com dados de evento, gravidade, período, riscos e recomendações.

## RF-04 — Importação de alerta INMET
O sistema deve importar alerta oficial por URL, gerar prévia e permitir confirmação.

## RF-05 — Territorialização
O sistema deve calcular municípios e regiões afetados com base na geometria do alerta.

## RF-06 — Edição e encerramento
O sistema deve permitir editar alertas e encerrar/cancelar conforme permissão de perfil.

## RF-07 — Visualização operacional
O sistema deve exibir painel com indicadores e status operacional dos alertas.

## RF-08 — Mapa multirriscos
O sistema deve apresentar mapa com filtros, KPIs e rankings territoriais.

## RF-09 — Módulo analítico
O sistema deve oferecer análises temporal, severidade, tipologia e índices.

## RF-10 — Auditoria
O sistema deve registrar ações de usuário em trilha histórica consultável.

## RF-11 — Gestão de usuários
O sistema deve permitir a administradores gerenciar contas, perfis e status.

## RF-12 — Exportações
O sistema deve gerar exportações operacionais em PDF, CSV e KML.

## RF-13 — Envio COMPDEC
O sistema deve permitir envio de alerta para contatos COMPDEC válidos.

## RF-14 — Consulta de histórico em PDF
O sistema deve gerar relatório PDF do histórico de ações filtradas.

## Referência
- Requisitos mapeados até **2026-03-28**.
